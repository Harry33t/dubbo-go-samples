方案一：构建本地「伪搜索」服务（最推荐，符合 Sample 调性）
不需要真实 API，而是自己造一个稍微复杂点的本地数据集，并实现查询逻辑。

做法：在后端服务中用内存（如 map 或 List）或者轻量级数据库（如 SQLite）存储几十条伪造的航班数据。

能力展示：Agent 能够根据用户的自然语言（如“帮我查一下明天从北京到上海的航班”），提取出 出发地=北京、目的地=上海、日期=明天 这三个参数，然后调用你的搜索函数，返回过滤后的结果，而不是像之前那样无论问什么都返回同一串死数据。

方案二：接入带有免费额度的第三方 API（需配置环境变量）
如果你觉得造假数据不够酷，想用真实数据。

做法：接入上一轮我们提到的有免费测试额度的接口（例如 Aviationstack 或天行数据）。

注意：在代码中绝对不能硬编码（Hardcode）你自己的 API Key。你需要把 API Key 提取到配置文件或环境变量中，并在 README 中注明：“若要测试真实航班搜索，请前往 XXX 申请免费 Key 并配置到环境变量中；若未配置，则降级使用本地 Mock 数据”。
